# Blender AI Controller (Ollama)

Control Blender with natural language using a **local Ollama model** — Copilot-style,
right inside the N-panel. No cloud, no API keys. Ask questions, plan tasks, or let an
agent build and edit your scene, with an optional per-step vision check and free
PolyHaven assets.

> ⚠️ **Security notice (read this).** In Agent mode this add-on executes Python code
> generated by a local LLM via `exec()`. A built-in **Safe scan** (on by default) blocks
> obvious dangerous calls (file/system/network), but it is **not a real sandbox**. Only
> run prompts you understand, review the AI Report, and use at your own risk.

---

## Features

- **Three modes** — **Ask** (answers only), **Plan** (todo plan, no execution),
  **Agent** (unlocks tools and builds/edits the scene).
- **Permission levels** (Agent only): `read` (plan only), `accept` (confirm each step),
  `auto` (fully automatic with self-fix).
- **Structured output** via Ollama JSON schema + low temperature for reliable plans.
- **Auto-retry**: failing steps are sent back to the model to fix and re-run.
- **Per-step vision check** (optional): after each step the AI picks a view angle
  (six faces / iso / free), a hidden camera auto-frames the edited object, and a
  screenshot is sent to a vision model to verify. Pauses on problems.
- **Beautify scene**: one click for EEVEE material preview + sky lighting + a sun + materials.
- **Procedural-first prompt**: nudges the AI to use Geometry Nodes + Shader nodes.
- **PolyHaven helpers**: the AI can call `ph_hdri("sunset")` and `ph_texture("wood floor")`
  to fetch free HDRIs and PBR materials.
- Scrollable AI report + todo list, **Force stop** button, inputs lock while running.
- Runs on the main thread via `bpy.app.timers`; background threads only call Ollama
  (never touches `bpy` off-thread).

## Requirements

- **Blender 4.1+** (loads on 4.2–5.x; a `blender_manifest.toml` is included for the
  Extensions platform).
- **[Ollama](https://ollama.com)** running locally (`ollama serve`, port `11434`).
- At least one model pulled. Recommended:
  | Model | ~size (q4) | Notes |
  |-------|-----------|-------|
  | `qwen2.5-coder:32b` | ~20 GB | Best bpy code accuracy; fits a 24 GB GPU |
  | `qwen3-coder:30b` | ~19 GB | Newer MoE, fast, tool-friendly |
  | `qwen3.6:27b` | ~17 GB | Multimodal (vision) — needed for the vision check |
- A **vision-capable model** (e.g. `qwen3.6`) is needed only for the per-step vision check.
- **Network** is used for Ollama (localhost) and PolyHaven downloads only.

## Install

Download this add-on as a **ZIP** (GitHub **Code → Download ZIP**, or a release asset).

- **Blender 4.2+**: drag the `.zip` into the Blender window, **or**
  **Edit → Preferences → Add-ons → Install from Disk…** and pick the `.zip`.
- **Blender 4.1**: **Edit → Preferences → Add-ons → Install…** and pick the `.zip`.

Then enable **"Blender AI Controller (Ollama)"**, press `N` in the 3D viewport → **AI** tab.

> Note: installing GitHub's auto-generated source zip names the module after the zip's
> top folder (e.g. `...-main`) — purely cosmetic. For a clean name, publish a Release whose
> zip contains a single `blender_ai_controller/` folder (see Packaging below).

## Usage

1. Press **Refresh** → confirm "Ollama: connected" and a model is selected.
2. Pick a **Mode** (and **Permission** for Agent), type a **Prompt**, press **Run**.
3. Watch the **AI Report** and **Todo** list. In `accept` you approve each step; on a
   failure or visual issue you choose **Skip / Cancel / Fix / Accept & continue**.
4. **Force stop** interrupts any time.

Tips: click **Beautify scene** to instantly upgrade lighting/materials on any scene;
keep **Prefer procedural nodes** and **Use PolyHaven** on for better results; split
complex requests into smaller prompts.

## Options

| Option | What it does |
|--------|--------------|
| Mode | Ask / Plan / Agent |
| Permission | read / accept / auto (Agent only) |
| Auto-retry | Times the AI auto-fixes a failing step |
| Safe scan | Block dangerous calls in AI code before running |
| Step vision check | Screenshot + vision-model review after each step |
| Prefer procedural nodes | Bias toward Geometry Nodes + Shader nodes |
| Use PolyHaven | Allow downloading free HDRIs / PBR materials |

---

## How it works

All `bpy` operations run on the main thread via a `bpy.app.timers` state machine
(plan → execute step → optional vision check → retry/fix → summary). Background threads
only do network I/O (Ollama, PolyHaven). This add-on talks to Ollama **directly** — it
does **not** use the MCP protocol.

---

## Limitations (honest)

Output quality is bounded by the local model. ~27–32B models do decent simple models but
limited refined shapes; **clean topology / edge flow is a weak spot** for code-driven
modeling. Lighting, materials, and procedural nodes are where AI helps most. The per-step
vision check is slower and local vision judgment is imperfect. Not a sandbox — see the
Security notice.

## Packaging (for maintainers)

This repository's contents **are** the add-on (`__init__.py` at the repo root). To build a
clean install zip, make sure no `__pycache__/` is present, then create a zip whose single
top-level folder is `blender_ai_controller/`. Example (Windows PowerShell, run from the
folder that contains this one):

```powershell
Compress-Archive -Path blender_ai_controller -DestinationPath blender_ai_controller.zip
```

Attach that zip to a GitHub Release.
